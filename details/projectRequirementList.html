Project Requirement List

Functional Requirements

  1. The customer should be able to upload the documents they want to be printed.
  2. The customer should be able to pick additional extra tasks on their documents (hole punch, binding, staple, etc.).
  3. The customer should be able to see a quote of how much their printing order will cost before confirming.
  4. Customers should be able to pay for their printing.
  5. The system should be able to notify the printing staff with an automated email when a printing order has been placed.
  6. The print shop worker should be able to send emails to a customer to inquire about their printing order.
  7. The print shop worker should be able to update the prices of various printing options (hole punch, binding, etc.)
  8. The customers should be able to fill out a text box to add a comment to their order to let the print shop worker know of any requests.
  9. The print shop worker should be able to view current orders.
  10. The print shop worker should be able to search for orders from a customer by name.
  11. The print shop worker should be able to update what the current status is on a specific order.
  12. Printshop workers should be able to cancel an order.
  13. Printshop workers should be able to upload a pricing sheet.
  14. The customers should be able to view a pricing sheet.
  15. The print shop workers should be able to create and edit articles.
  16. The print shop worker should be able to remove an article.
  17. The customers should be able to view article previews on the home page.
  18. The customers should be able to view an article.

Non-Functional Requirements

  1. The system should be supported on different browsers.
  2. The system should be supported on mobile.
  3. The systems transactions should be encrypted with SSL
  4. The system should provide refunds to customers upon order cancellation.
  5. The system should be able to store pricing information on DynamoDB.
  6. The system should be able to store orders on DynamoDB.
  7. The system should be able to calculate the cost of printing based on: number of pages, the colour of paper, additional extras (hole punch, binding, etc.) reliably in under 2 seconds.
  8. The system should be able to store articles on DynamoDB.
  9. Improve/migrate ITSA site to the system we use for hosting backend.

Final System vs Initial Requirements

The final version of the ITSA Print shop met almost all of the requirements set out at the beginning of the project. Certain requirements were changed, removed or expanded upon as detailed in the section above. Most functionalities remained the same for the print shop other than customer accounts being removed. Other additions to the system came from the need for quality of life improvements both for customers and print shop workers, such as automated email for both the admins and customers.

The decision was also made to redesign the main ITSA website, which added multiple new requirements, such as the addition of articles. When compared against the original 21 requirements submitted, the system has met 14 of the initial requirements. 3 out of 6 of the nice to have requirements were implemented, and 1 out of 1 stretch goal was implemented. The requirements not present are mostly related to customer account functionality. The final list of requirements reflect a better state of the system as many additions as well as a few removals, bringing the final total requirements up to 28 between the functional and non-functional requirements
